0. Introduction.

In the past few lectures, we’ve discussed how to implement a Lisp evaluator in OCaml. For this laboratory assignment, you will modify the Lisp evaluator so it can evaluate three kinds of expressions that it could not evaluate before. Among other things, this gives you some practice reading and modifying code that you didn’t write yourself—as you might need to do in the Real World.

1. Implementation.

The file tests11.ml contains OCaml source code for a Lisp evaluator like the one discussed in the lectures. You must modify the evaluator so it can evaluate three new primitive functions: atom, imply, and let. You must do it by adding three new calls to the OCaml function primitive. The new primitives are described as follows, where the arrow ‘⇒’ means returns.

Atom. In Lisp, an atom is an object that can’t be split into smaller parts. (Scientists once believed that about real atoms, too.) The subset of Lisp discussed in the lectures has five kinds of atoms: closures, nil’s, numbers, primitives, and symbols.
      The function is-atom takes one argument. It returns the symbol t (true) if its argument evaluates to an atom, and nil (false) otherwise. Here are some examples of how is-atom must work.

(is-atom nil)

  ⇒  

t

(is-atom (quote a))

  ⇒  

t

(is-atom (quote (a b c)))

  ⇒  

nil

(is-atom 7734)

  ⇒  

t

(is-atom is-atom)

  ⇒  

t

(is-atom (+ 2 2))

  ⇒  

t

If is-atom is called with no arguments, or with more than one argument, then it must raise the exception EvaluatorError with a descriptive string as an error message. Modify the Lisp evaluator so it can evaluate calls to is-atom.

Imply. In propositional logic, α → β is called an implication, and is read as ‘‘alpha implies beta.’’ It’s an abbreviation for (¬ α) ∨ β. If α is false, then α → β is true. If β is true, then α → β is true. Otherwise α → β is false.
      The Lisp function (imply e₁ e₂ ⋯ eₙ) acts like a series of one or more implications e₁ → e₂ ⋯ → eₙ. It’s like an or, where all arguments but the last are inside not’s: (or (not e₁) (not e₂) ⋯ eₙ).
      The function imply evaluates its n arguments e₁, e₂ ⋯, eₙ one at a time, from left to right. If one of the first n−1 arguments evaluates to nil, then imply stops immediately, and returns t without evaluating the remaining arguments. Otherwise, it returns the result of evaluating the nth argument. Here are some examples of how imply must work.

(imply t)

  ⇒  

t

(imply nil)

  ⇒  

nil

(imply nil (/ 0 0))

  ⇒  

t

(imply t nil (/ 0 0))

  ⇒  

t

(imply t t t 100)

  ⇒  

100

(imply (quote a) (quote b) (quote c) (quote z))

  ⇒  

z

If imply is called with no arguments, then it must raise the exception EvaluatorError with a descriptive string as an error message. Modify the Lisp evaluator so it can evaluate calls to imply.

Let. The Lisp function (let s e₁ e₂) acts something like an OCaml let-expression. First, let evaluates the expression e₁. Next, it binds the symbol s in the local environment to the result of evaluating e₁. Then it evaluates the expression e₂, which may contain at least one appearance of s. Finally, let restores the current environment to what it was before s was bound. Here are some examples of how let must work.

(let n 1 n)

  ⇒  

1

(let n 2 n)

  ⇒  

2

(let two 2 (+ two two))

  ⇒  

4

(let two 2 (is-atom two))

  ⇒  

t

(let a 1 (let b 2 (+ a b)))

  ⇒  

3

If let is called with fewer than three arguments, or more than three arguments, then it must raise the exception EvaluatorError. If its first argument is not a symbol, then it must also raise the exception EvaluatorError. In either case, the exception must have a descriptive string as an error message. Modify the Lisp evaluator so it can evaluate calls to let. Hint: use the OCaml function envPut to create a new environment in which s is bound, and and pass it to the OCaml function evaluating.
